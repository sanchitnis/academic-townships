# Academic Townships Charter Evolution Loop (v0.1)

This file defines the recurring, auditable loop used to evolve the project charter through committee, external experts, and AI collaboration.

## Execution mode
**Mode 1 (IDE-first):** run critique and synthesis work directly through repository workflows and pull requests.

## Six-step sprint cycle
1. **Sprint Planning Gate**
   - Set sprint goal and target tension(s).
   - Assign committee reviewer(s), expert invitees, and AI model roles.
2. **Critique Wave**
   - Generate critiques from multiple lenses (policy, governance, urban systems, industry integration, inclusion).
   - Save critiques in `docs/charter_evolution/critiques/`.
3. **Aggregate & Synthesize**
   - Produce `SYNTHESIS_WAVE_N.md` with convergences and unresolved tensions.
4. **Polycentric Arbitration**
   - Committee + external experts resolve what can be resolved.
   - Preserve irreducible tensions in `docs/governance/dissensus_ledger.md`.
5. **Draft Proposal & Validation Wave**
   - Draft consolidated charter changes.
   - Re-run adversarial validation on the draft.
6. **Lock, Tag, Retrospective**
   - Promote accepted draft as canonical version.
   - Update memory docs and process improvements.

## Mandatory gates
- **Gate 1:** Synthesis approval by committee.
- **Gate 2:** Arbitration output approval.
- **Gate 3:** Draft proposal approval.
- **Gate 4:** Validation approval.
- **Gate 5:** Final lock/tag signoff.

## Core files touched each sprint
- `memory/project-state.md`
- `docs/charter_evolution/critiques/README.md`
- `docs/charter_evolution/SYNTHESIS_WAVE_N.md`
- `docs/governance/dissensus_ledger.md`
- `memory/decision-registry.md`
- `memory/failure-registry.md`
